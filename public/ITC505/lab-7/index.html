Space Adventure Mad Lib

Addendum

During the implementation of this Mad Lib project, I encountered the following challenges and resolved them as follows:

  • Handling Form Data on the Server Side: At first, I was able to receive the form data with some complications in the storage and handling of the values. I fixed this by ensuring that I used express.urlencoded({ extended: true }), form data that is received is then passed.
  • Route Conflict with Static Files: Static files also posed a problem together with defining custom routes in my server.js. I was able to rectify this by making sure that the static file routes were created before the custome routes.
  • Preventing Empty Form Submissions: At first, I failed to handle blank forms correctly, which led to half-filled Mad Libs. I incorporated the form validation in the back end side in a way that none of the fields are left empty when the output is generated.
  • Displaying Mad Lib Output Correctly: The issue was with string concatenation and as a result the output was coming out all mixed up. This I was able to solve by using string interpolation and also ensuring good HTML planning in terms of structure.
